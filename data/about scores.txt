Yes, they matter for group-phase rows — and here's the exact rule from the code:

simulation.py:37 only picks up a row as a completed match if phase == "group" and group is a valid letter (A–L). If either is wrong, the match gets treated as unplayed and re-simulated.

For knockout rows (r32, r16, qf, sf, final) — the file loads them without error, but the simulation currently ignores them. It only uses group-phase results to build standings, then simulates the entire knockout bracket fresh. So those rows have no effect for now.
